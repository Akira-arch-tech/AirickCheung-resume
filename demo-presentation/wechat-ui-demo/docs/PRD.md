# 微信核心功能链路 PRD（UI Demo 对标用）

| 项目 | 说明 |
|------|------|
| 文档用途 | 梳理**真微信**的核心用户链路，作为本仓库「微信风格 UI Demo」迭代的需求基线 |
| 产品形态 | 纯前端界面演示，**无真实 IM / 支付 / 登录**，数据均为 Mock |
| 版本 | v0.1 |
| 更新日期 | 2026-04-06 |

---

## 1. 背景与目标

### 1.1 背景

微信是「即时通讯 + 社交关系 + 内容消费 + 服务入口」的复合体。本 Demo 用于作品集展示：**在浏览器中复刻关键界面与交互路径**，体现对复杂 C 端产品信息架构与状态流转的理解。

### 1.2 目标

- **对齐认知**：把「用户从打开到完成核心任务」的链路写清楚，避免只做零散页面。
- **指导实现**：每一条链路可映射为 Demo 中的页面、状态与 Mock 数据。
- **边界清晰**：明确哪些能力仅做壳层交互、哪些永久 Out of Scope，避免法律与品牌风险。

### 1.3 非目标

- 不实现真实消息投递、语音视频通话、支付、小程序运行时。
- 不采集用户隐私数据；不仿冒官方应用商店包名或商标用于分发。

### 1.4 合规声明（产品约束）

- 本 Demo **与腾讯微信无关联**，须在界面或文档中保留「界面练习 / 演示」说明。
- 对外展示时避免让人误以为可获得真实微信服务。

## 2. 用户与典型场景

| 角色 | 描述 | 典型场景 |
|------|------|----------|
| 普通用户 | 已登录，有好友与群 | 收发信息、刷朋友圈、改个人资料 |
| 群成员 | 在多个群内 | 群聊、@、群公告、群工具入口 |
| 内容消费者 | 订阅公众号 / 视频号（真产品） | 发现页进入内容流（Demo 可占位） |

Demo 默认假设：**已处于主界面（四 Tab）**，不单独做启动屏与短信登录流程（可作为后续增强项）。

---

## 3. 信息架构（IA）总览

真微信主结构可抽象为：

```text
微信（会话列表）
├── 单聊 / 群聊会话
│   ├── 消息流（文本 / 图片 / 语音 / 视频 / 文件 / 链接 / 位置 / 红包 / 转账 / 名片…）
│   ├── 输入区（键盘 / 语音按住说 / 表情 / 更多面板）
│   └── 会话级能力（置顶、免打扰、查找聊天内容、清空…）
├── 通讯录
│   ├── 新朋友、群聊、标签、公众号（等）
│   └── 联系人详情 → 发消息 / 音视频 / 资料
├── 发现
│   ├── 朋友圈
│   ├── 视频号、直播、扫一扫、购物、游戏、小程序（等）
└── 我
    ├── 个人信息与服务
    ├── 设置（账号、通用、隐私…）
    └── 支付入口（真产品强相关，Demo 仅壳）
```

## 4. 核心功能链路（按链路拆解）

以下每条链路采用统一结构：**用户目标 → 前置条件 → 主路径步骤 → 关键界面/状态 → Demo 建议深度**。

---

### 4.1 账号与会话入口（冷启动链路）

| 项 | 内容 |
|----|------|
| **用户目标** | 进入可使用的主界面 |
| **前置条件** | 已安装客户端；真产品需登录态 |
| **主路径** | 启动 App →（可选）解锁 / 生物识别 → 进入「微信」Tab 会话列表 |
| **关键界面** | 启动、登录/注册、验证、主框架 |
| **Demo 建议** | **P2**：可做「假登录」一页后进入主框架；当前可省略，默认直达主框架 |

---

### 4.2 消息：会话列表链路

| 项 | 内容 |
|----|------|
| **用户目标** | 找到某个聊天并进入 |
| **主路径** | 打开微信 Tab → 浏览列表（置顶在上、时间排序）→ 点击一行进入会话 |
| **分支** | 搜索会话 / 未读角标 / 标为未读 / 删除或隐藏会话（真产品能力多样） |
| **关键状态** | 未读数、最后一条摘要、置顶标记、免打扰（小月亮） |
| **Demo 建议** | **P0**：列表 + 搜索过滤 + 未读 + 置顶；**P1**：左滑菜单（标未读/删除）动画壳 |

---

### 4.3 消息：单聊会话链路

| 项 | 内容 |
|----|------|
| **用户目标** | 与对方完成一轮沟通 |
| **主路径** | 进入会话 → 查看历史 → 输入文字 → 发送 → 对方回复出现在时间流中 |
| **分支** | 长按消息（复制/撤回/引用/多选…）、发送失败重试、图片预览、语音播放 |
| **关键界面** | 导航栏（返回、标题、更多）、消息区、输入条 |
| **Demo 建议** | **P0**：文本发送 + 滚动到底部 + 时间气泡；**P1**：多类型气泡占位、长按菜单壳；**P2**：引用回复样式 |

---

### 4.4 消息：群聊会话链路

| 项 | 内容 |
|----|------|
| **用户目标** | 在群内同步信息、@某人、查看成员 |
| **主路径** | 进入群会话 → 发消息 →（可选）@成员 → 消息带头像/昵称区分 |
| **分支** | 群公告、群待办、群工具、邀请确认（真产品） |
| **Demo 建议** | **P0**：与单聊共用组件，标题为群名；**P1**：系统消息样式（「xxx 邀请 xxx」Mock）；**P2**：@ 选择器壳 |

---

### 4.5 消息：输入与「更多」发送链路

| 项 | 内容 |
|----|------|
| **用户目标** | 用非纯文本方式表达信息 |
| **主路径** | 点「+」→ 选择相册 / 拍摄 / 语音输入 / 位置 / 红包等 → 进入对应子流程 → 发送后在会话内展示 |
| **关键子链路（摘要）** | 相册选图 → 预览 → 发送；按住说话 → 松开发送语音条；位置 → 选点 → 地图卡片 |
| **Demo 建议** | **P0**：+ 面板网格 + 各类型**假发送**（Mock 气泡）；**P1**：语音条「播放」动画、图片 lightbox 壳 |

---

### 4.6 消息：表情与贴纸链路

| 项 | 内容 |
|----|------|
| **用户目标** | 快速插入表情或整套表情 |
| **主路径** | 点表情入口 → 选最近/emoji/收藏 → 点击上屏 |
| **Demo 建议** | **P0**：快捷 emoji 条（已实现可保留）；**P1**：半屏表情面板 Tab 壳 |

### 4.7 会话详情与设置链路（右上角「…」）

## 5. 关键用户旅程（端到端文字流）

### 旅程 A：收到新消息后回复

1. 用户在微信 Tab 看到会话预览更新与未读角标。  
2. 进入会话，消息区定位到最新未读附近（真产品逻辑复杂，Demo 可简化为滚底）。  
3. 输入并发送，列表预览与未读状态更新。

### 旅程 B：从通讯录发起聊天

1. 通讯录找到联系人 → 详情 → 发消息。  
2. 若已有会话则进入该 thread；若无则 Demo 可提示「可扩展新建会话」。

### 旅程 C：刷朋友圈并点赞

1. 发现 → 朋友圈 → 浏览 Feed → 点赞/取消赞。  
2. （扩展）点评论进入评论列表。

### 旅程 D：聊天中发一张图

1. 会话内点 + → 相册 → 选图（Demo 用预设图或 Emoji 占位）→ 发送。  
2. 会话内出现图片气泡，列表预览显示 `[图片]`。

## 6. 非功能需求（Demo 级）

| 类型 | 要求 |
|------|------|
| 性能 | 首屏可接受；长列表会话/消息使用虚拟滚动为 **可选优化** |
| 适配 | 移动优先；外框在桌面端居中展示即可 |
| 无障碍 | 主要按钮与输入框有可读 label；列表项建议 `aria-label`（部分已做） |
| 部署 | 静态托管（GitHub Pages），构建 `base` 与仓库路径一致 |
| 可维护性 | Mock 数据集中；链路对应页面组件边界清晰 |
